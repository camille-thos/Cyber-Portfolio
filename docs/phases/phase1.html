Phase 1 — Mise en place de l’infrastructure

Cette phase consiste à construire une infrastructure Active Directory fonctionnelle dans un environnement réseau segmenté reproduisant un système d'information d’entreprise.

Objectif

Mettre en place une infrastructure Active Directory fonctionnelle servant de base aux futures phases d’observabilité, détection et réponse.

  • Firewall périmétrique
  • Réseau interne segmenté
  • Contrôleur de domaine
  • Poste client joint au domaine
  • Machine attaquante isolée

Aucun mécanisme de détection ou SIEM n’est utilisé à ce stade.

Architecture

Le réseau interne représente le système d’information tandis que le réseau ATTACK simule une zone non fiable utilisée pour les attaques.

Organisation Active Directory

L’Active Directory est structuré pour représenter une organisation d’entreprise classique et faciliter les scénarios d’analyse.

  • OU_Comptabilite
  • OU_IT
  • OU_RH

Chaque OU contient :

  • Utilisateurs
  • Groupes
  • Ordinateurs

Cette organisation permet l’application de GPO ciblées, la simulation de délégations et des scénarios réalistes d’élévation de privilèges.

Objectif de validation

  • ✅ Domaine Active Directory opérationnel
  • ✅ Client joint au domaine
  • ✅ Communication réseau fonctionnelle
  • ✅ Segmentation réseau active

Les captures d’écran seront enrichies progressivement.
